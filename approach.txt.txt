Traffic Demand Prediction – Approach

 Problem Overview

The goal is to predict traffic demand using historical traffic, weather, and location-based features. The evaluation metric is R² score.

Data Preprocessing

* Loaded train and test datasets.
* Identified target variable: demand.
* Extracted time-based features from timestamp:

  * hour
  * minute
  * peak-hour indicator
* Handled categorical variables using Label Encoding.

Feature Engineering

* Converted timestamp into numerical features.
* Encoded categorical variables:

  * geohash
  * road type
  * weather
  * large vehicles
  * landmarks

 Model Used

* Random Forest Regressor was used as baseline model due to its robustness on tabular data.

Tools Used

* Python
* Pandas
* Scikit-learn
* Jupyter Notebook

 Future Improvements

* Use gradient boosting models like XGBoost or LightGBM
* Improve geospatial feature encoding
* Advanced feature engineering on traffic patterns
